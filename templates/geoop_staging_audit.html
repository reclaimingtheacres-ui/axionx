What was verified in dev vs. what must be checked live
Verified in dev (Replit):
  • Route /admin/geoop-staging-audit is admin-only, read-only, and returns correct counts.
  • Matching logic uses import_status='imported' and axion_note_id / axion_doc_id fields from the actual table schemas.
  • CSV exports for unmatched notes, unmatched files, and broken references work correctly.
  • Server logs are written on each audit run (user, timestamp, counts).
  • No delete, modify, archive, or migration functions exist on this page.
Must be checked live by Grant (axionx.com.au):
  • The actual counts — staging tables only exist on the production database.
  • Whether unmatched records are genuine gaps or expected pending items.
  • Whether broken references point to jobs that were deleted or never linked.
  • File paths / blob paths for unmatched files — whether they still exist in Azure or on disk.
  • Sign-off on any counts before any cleanup is considered (outside scope of this tool).
