Step 1: Stage CSV Data

Upload the GeoOp jobs.csv and notes.csv exports. Data is loaded into staging tables for review before importing. The Description field is parsed to extract client, account, regulation type, amounts, security details, registration, VIN, and delivery instructions.

Attachment Scan (Local): If the GeoOp attachments/ folder has been uploaded to the server, enter the path below to scan and index files by Job ID and Note ID. Structure: attachments/{job_id}/{note_id}/{file_id}/{filename}

Attachment Scan (Azure Blob): Provide an Azure Blob Storage container SAS URL to stream attachments directly from the cloud. Files are matched against imported notes by Job ID and Note ID from the blob path: attachments/{job_id}/{note_id}/{file_id}_{filename}. Zip archives are extracted automatically. Only files matching already-imported notes are linked — no notes are overwritten.

Step 2: Review Diagnostics

Click "Load Diagnostics" to generate a report, or "View Rejects" to see skipped notes.

Step 3: Execute Import

Import staged data into AxionX. Jobs, customers, security items, and optionally notes will be created. Existing jobs matched by reference number or GeoOp Job ID will be skipped (insert-only mode) or updated.

Backfill GeoOp Descriptions

Restores the original GeoOp Description text to all previously imported jobs. Creates a legacy import note on each job, preserves the raw source text in a permanent field, and re-parses structured data (Reg, VIN, arrears, costs, next payment, regulation status, security details). Safe to run multiple times — existing notes and user-edited fields are not overwritten. Processes in batches of 250 with progress checkpointing. Can be restarted safely if interrupted.
